# fe-dev:ui Skill Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add design-to-code capability to the fe-dev plugin, enabling systematic conversion of MasterGo design mockups to Vue3 + Element Plus pages.

**Architecture:** Three new skills (ui, ui-add, ui-gen) with shared utilities (ui-utils.md) and templates (ui-pages.json, design-spec.md). ui-add analyzes design mockups via MasterGo MCP and produces structured specs; ui-gen generates Vue pages using project context (types, services) following a strict styling hierarchy (Element Plus > Tailwind > Scoped SCSS). The ui skill routes to list/update/check subcommands.

**Tech Stack:** Claude Code Skills (SKILL.md), MasterGo MCP, Vue3, Element Plus, Tailwind CSS, SCSS

**Spec:** `docs/superpowers/specs/2026-03-20-fe-dev-ui-design.md`

### Task 5: Create ui SKILL.md (entry router)

**Files:**
- Create: `fe-dev/skills/ui/SKILL.md`

- [ ] **Step 1: Create ui SKILL.md**

Frontmatter:
```yaml
---
name: ui
description: UI 设计稿管理。触发词: "ui", "ui list", "设计稿列表", "ui check", "ui update", "设计稿更新"
---
```

Body: Command router (same pattern as spec SKILL.md):

| Subcommand | Description |
|------------|-------------|
| (none) | Default to list |
| `list` | Show all pages with status |
| `update <page-id>` | Diff-based design update |
| `check [page-id]` | Quality check on generated code |

**list** subcommand:
1. Get feat name, read ui-pages.json
2. Format as table: id | name | status | target | mastergo
3. If no pages, suggest running `/fe-dev:ui-add`

**update** subcommand:
1. Get feat name, read existing spec for page-id
2. Fetch new design data (same fallback as ui-add)
3. AI compare old/new, generate diff report (layout, tokens, components, fields, interactions)
4. AskUserQuestion: accept all / select items / cancel
5. Update spec + ui-pages.json (status: spec-done, updatedAt)
6. Prompt to re-run ui-gen

**check** subcommand:
1. Get feat name, read ui-pages.json
2. For each page with status=converted (or specified page-id), read .vue file
3. Scan against quality rules from ui-utils.md
4. Output issue list sorted by severity (error/warning/info)
5. No errors -> update status to reviewed

- [ ] **Step 2: Commit**

```bash
git add fe-dev/skills/ui/SKILL.md
git commit -m "feat(ui): add ui skill entry router with list/update/check"
```
