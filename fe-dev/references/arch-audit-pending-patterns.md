# AI 代码体检 - 待观察反模式池

> 由 `/fe-dev:arch-audit` 在审计过程中**自动捕获**的"疑似新反模式"。
>
> **状态**：pending（待观察）。命中数累计中，**不进入修复列表**，仅在报告中作为"提示性匹配"标注。
> **晋升路径**：月度评审，命中次数 ≥ 3 且误报率低的条目，迁移到 `arch-audit-checklist.md` 转正为 known，编号 `P-NNN` → `M-NNN`。
> **淘汰路径**：命中数长期 0 或误报率 >50%，标记 retired 或直接删除。

## 编号规则

- 格式：`P-NNN`，从 P-001 起递增
- 编号不复用，删除后下一条用新编号

## 字段说明

每个待观察条目包含：

```markdown
### [P-NNN] <简短标题>

- **建议归类**：维度 N (<维度名>)
- **建议严重度**：critical / major / minor
- **首次发现**：YYYY-MM-DD
- **命中次数**：N
- **模式描述**：<一段话说明 LLM 在什么情况下会写出这种代码>
- **检测要点**：<下次怎么找>
- **样本**：
  - <文件路径:行号>（首次发现位置）
  - <若有多次命中，列出 2-3 个代表性样本>
- **晋升评估**：<空，由月度评审时填写>
```
