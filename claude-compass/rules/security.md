# 安全规范

## 要求

- 禁止在代码中硬编码密钥、密码、Token 等敏感信息
- 所有用户输入必须校验和转义，防止 XSS 和 SQL 注入
- API 接口必须进行权限校验，禁止未授权访问
- 敏感数据（密码、手机号等）存储和传输时必须加密或脱敏
- 日志中禁止打印敏感信息（密码、Token、身份证号等）
- .env、credentials.json 等配置文件必须加入 .gitignore

## 原因

安全漏洞一旦上线，修复成本远高于预防成本。这些规则覆盖 OWASP Top 10 中最常见的风险。
